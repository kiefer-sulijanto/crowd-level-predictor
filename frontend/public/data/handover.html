SIM-DAC Handover Document

Crowdedness Prediction System for Food Courts

Project: MakanMap Crowdedness Predictor

Date: 14 January 2026

Project Overview

Objective: Develop a machine learning system to predict crowdedness levels at various food court locations in Singapore.

Business Value: Help users plan their visits to avoid peak hours, optimize food court operations, and provide real-time crowdedness insights.

Success Metrics:

  • Prediction accuracy > 85%
  • Model inference time < 100ms
  • System uptime > 99%

Dataset Information

Generated Mock Data

We've generated synthetic data that mimics real-world QR code scanning patterns at 20 food court locations across Singapore.

Important Notes:

  • Data spans from Jan 1, 2025 to Jun 30, 2025
  • Operating hours: 6 AM to 10 PM daily
  • Weather data is synthetic but informed by real Singapore API data
  • Dirty dataset contains 13 types of data quality issues for testing cleaning pipelines
  • Aggregation of data is allowed for feature engineering/creating new features Eg: Grouping scan records to 15 minute timeframes

Data Cleaning Pipeline

Before model development, the team must create a robust data cleaning pipeline to handle:

Data Quality Issues Present:

  • Missing values (5%) - Random NaN values
  • Outliers (3%) - Extreme values in numeric columns
  • Duplicates (2%) - Duplicate rows
  • Data type errors (4%) - Numeric values converted to strings
  • Inconsistent casing (10%) - Mixed cases in text columns
  • Timezone mix (2%) - Timezone annotations added
  • Unit mismatches (3%) - Temperature in Fahrenheit
  • Geographic errors (2%) - Swapped lat/long coordinates
  • Logical errors (4%) - Inconsistent day_of_week and is_weekend
  • Column rename issues (30% chance) - Random column names
  • Whitespace issues (15%) - Extra spaces in text
  • Special character errors (8%) - Random special characters
  • Operating hour violations (1%) - Records outside 6 AM - 10 PM

Exploratory Data Analysis (EDA) Requirements

Perform comprehensive EDA to understand patterns and inform feature engineering:

  • Temporal Analysis: Hourly, daily, weekly patterns
  • Location Analysis: Busiest locations, geographic patterns
  • Weather Impact: Correlation between weather and crowdedness
  • Holiday Effects: Impact of public holidays and weekends
  • Cross-feature Analysis: Interactions between time, location, weather
  • Anomaly Detection: Identify unusual patterns
  • Seasonality Analysis: Monthly and seasonal trends

Create visualizations for:

  • Heatmaps of crowdedness by time and location
  • Time series decomposition
  • Correlation matrices
  • Geographic distribution maps
  • Feature importance plots
  • Extra visualizations are also welcome!

Machine Learning Tasks

We need to explore multiple approaches to predict crowdedness. The types of models used are up to the team's discretion:

Classification Approach

Target: Predict crowdedness level (Low, Medium, High)

Models to try:

  • Random Forest Classifier
  • XGBoost Classifier
  • LightGBM Classifier
  • Neural Network (MLP)
  • Ensemble methods

Metrics: Accuracy, Precision, Recall, F1-Score, ROC-AUC

Regression Approach

Target: Predict number of scans or crowdedness score (0-1)

Models to try:

  • Gradient Boosting Regressor
  • Random Forest Regressor
  • XGBoost Regressor
  • Support Vector Regressor
  • Neural Network Regression

Metrics: MAE, RMSE, R², MAPE

Time Series Approach

Target: Forecast future crowdedness patterns

Models to try:

  • ARIMA/SARIMA
  • Prophet
  • LSTM/GRU Networks
  • Time Series Forests
  • Transformer-based models

Metrics: SMAPE, MASE, Forecast Accuracy

Production Requirements

Real-time Prediction System:

  • API Endpoints: REST API for predictions
  • Model Serving: FastAPI or Flask deployment
  • Monitoring: Model performance tracking
  • Retraining: Automated retraining pipeline
  • A/B Testing: Framework for model comparison

Infrastructure Requirements:

  • Data Pipeline: Extracting data from the database will be handled by AAQT, the team may start with the CSV file.
  • Model Registry: MLflow for model tracking and monitoring
  • Containerization: Docker for deployment
